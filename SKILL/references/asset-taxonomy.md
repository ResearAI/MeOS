# Asset Taxonomy

## Stable asset families

- `assets/work/`
  - technical standards
  - acceptance rules
  - architecture preferences
- `assets/thought-style/`
  - decomposition habits
  - contract reasoning
  - validation stance
- `assets/workflow/`
  - planning routines
  - implementation routines
  - audit routines
  - verification routines
- `assets/principles/`
  - trade-offs
  - risk boundaries
  - first-principles rules
- `assets/taste/`
  - UI taste
  - product taste
- `assets/knowledge/`
  - semantic
  - episodic
  - procedural
- `assets/preferences/`
  - input
  - output
  - interests
- `assets/corrections/`
  - explicit overrides

## Promotion guidance

Promote into:

- `work/` when the rule affects technical execution across tasks
- `thought-style/` when the rule changes how problems are framed
- `workflow/` when the rule is a repeatable sequence
- `principles/` when the rule is a stable trade-off or boundary
- `taste/` when the rule changes UI or product judgment
- `knowledge/` when the rule is factual or procedural background
- `preferences/` when the rule is mainly about style or favored presentation
- `corrections/` when the user explicitly overrides prior understanding

## Complementary graph rule

Assets are the human-readable durable layer.

Person-graph claims are the structured complementary layer:

- claims live in `evidence/`
- graph projections live in `runtime/graph/`
- assets remain the canonical prose representation when prose is useful
